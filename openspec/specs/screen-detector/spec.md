# screen-detector

> 屏幕检测器 capability：在 Web 端为显示器提供"侧边栏 + 主画布 + 控制条"的目视检测面板集合，覆盖纯色、线条、干扰图案、对比度矩阵、渐变、饱和度、屏幕信息、多点触控等约 25 项检测。

## Requirements

### Requirement: 应用注册与入口

系统 SHALL 在 `apps/web/src/apps/index.ts` 的 `appRegistry` 中注册一个 ID 为 `screen-detector`、名称为"屏幕检测器"、图标为 `Monitor`、分类标签为"实用"的应用条目，并以懒加载方式引入 `./screen-detector`。

#### Scenario: 从应用市场打开屏幕检测器

- **WHEN** 用户进入 apps 列表页并在"实用"分类下点击"屏幕检测器"卡片
- **THEN** 系统 MUST 通过 `React.lazy` 加载 `screen-detector` 入口组件并渲染欢迎页

### Requirement: 三段布局

系统 SHALL 采用"左侧侧边栏 + 顶部控制条 + 主画布"的布局：侧边栏列出全部检测项并按分类分组；顶部控制条显示当前项名称、用途说明、"全屏显示"、"结束测试"按钮；主画布渲染当前检测项内容。

#### Scenario: 默认进入欢迎页

- **WHEN** 用户首次进入屏幕检测器
- **THEN** 系统 MUST 在主画布渲染"欢迎页"：项目介绍 + 当前分辨率（视口 / 屏幕 / 颜色深度 / DPR）+ 光敏警告

#### Scenario: 切换检测项

- **WHEN** 用户在侧边栏点击任一检测项
- **THEN** 系统 MUST 立即切换主画布到该检测项，并更新控制条的标题与描述

### Requirement: 检测项清单

系统 SHALL 在侧边栏按以下顺序与分组提供至少 23 个检测项（与参考站对齐），并补充屏幕信息与多点触控 2 项：

- **介绍**：欢迎页
- **纯色**（8）：纯红色、纯绿色、纯蓝色、纯黄色、纯青色、纯洋红色、纯白色、纯黑色
- **线条**（6）：水平线 1、水平线 2、垂直线 1、垂直线 2、正斜线 1、正斜线 2
- **干扰图案**（5）：点阵、微型图案、闪烁方块、方形网格、彩色网格
- **对比度**（2）：对比度(暗)、对比度(亮)
- **渐变**（4）：白色渐变、红色渐变、绿色渐变、蓝色渐变
- **饱和度**（1）：饱和度
- **现代补充**（2）：屏幕信息、多点触控
- **结束**：测试结束

#### Scenario: 分组显示

- **WHEN** 用户查看侧边栏
- **THEN** 系统 MUST 按上述分组渲染标题与项目列表，分组之间有可视分隔

### Requirement: 全屏模式

系统 SHALL 提供"全屏显示"按钮，调用浏览器 Fullscreen API 将主画布容器全屏化；侧边栏与控制条不进入全屏元素，从而被自动隐藏。

#### Scenario: 进入全屏

- **WHEN** 用户点击"全屏显示"按钮
- **THEN** 系统 MUST 对主画布容器调用 `requestFullscreen()`（必要时使用 webkit 前缀回退）

#### Scenario: 退出全屏

- **WHEN** 用户在全屏模式下按 `Esc` 或 `F`，或点击"结束测试"
- **THEN** 系统 MUST 调用 `document.exitFullscreen()` 并恢复三段布局

#### Scenario: 降级伪全屏

- **WHEN** 浏览器拒绝或不支持 Fullscreen API
- **THEN** 系统 MUST 改用 CSS `position: fixed; inset: 0; z-index` 伪全屏，并在控制条上提示"已使用兼容模式"

### Requirement: 全屏内无干扰自动隐藏

系统 SHALL 在全屏模式下，1.5 秒无键鼠操作后自动淡出控制条；鼠标移动或按键时重新显现 1.5 秒；按 `H` 键可强制隐藏/显现，状态持续直到再次按 `H`。

#### Scenario: 自动淡出

- **WHEN** 用户在全屏模式下 1.5 秒内无任何键鼠操作
- **THEN** 控制条与所有覆盖层 MUST 完全淡出，画布完全是检测内容

#### Scenario: 强制隐藏

- **WHEN** 用户按下 `H`
- **THEN** 控制条 MUST 立即隐藏并不再因鼠标移动显现，直到再次按 `H`

### Requirement: 键盘与点击切换

系统 SHALL 支持以下键鼠交互：

- `←` / `→`：上一项 / 下一项检测项（按侧边栏顺序）
- `F`：进入/退出全屏
- `Esc`：退出全屏
- 鼠标单击主画布空白处：下一项

#### Scenario: 方向键切换

- **WHEN** 用户按下 `ArrowRight`
- **THEN** 系统 MUST 切换到侧边栏中下一个检测项；到达末尾时回到首项

#### Scenario: 点击切换

- **WHEN** 用户在画布空白处单击
- **THEN** 系统 MUST 切换到下一个检测项

### Requirement: 自动巡检

系统 SHALL 提供"自动巡检"模式：按固定时间间隔自动切换到下一检测项；按 `Space` 启动/暂停；按 `[` / `]` 在 1s / 2s / 5s 三档间隔之间切换。

#### Scenario: 启动巡检

- **WHEN** 用户按下 `Space`
- **THEN** 系统 MUST 按当前间隔（默认 2s）自动切换到下一个检测项；再次按 `Space` 暂停

#### Scenario: 调整间隔

- **WHEN** 用户按下 `[` 或 `]`
- **THEN** 系统 MUST 在 1s / 2s / 5s 之间切换并在角落短暂提示当前间隔

### Requirement: 纯色检测面板（坏点检测核心）

系统 SHALL 为以下 8 种颜色各提供一个独立面板，全屏时画布为该颜色 100% 填充，无任何非透明叠加元素（控制条须可被彻底隐藏）：红 `#ff0000`、绿 `#00ff00`、蓝 `#0000ff`、黄 `#ffff00`、青 `#00ffff`、洋红 `#ff00ff`、白 `#ffffff`、黑 `#000000`。

#### Scenario: 全屏纯色填充

- **WHEN** 用户进入任一纯色面板并进入全屏
- **THEN** 主画布 MUST 100% 填充该 HEX 颜色，无边框、无文字（控制条已淡出后）

#### Scenario: 在非全屏下显示色值标签

- **WHEN** 用户处于非全屏纯色面板
- **THEN** 控制条 MUST 显示该颜色的中文名与 HEX 值（如 "纯红色 #ff0000"）

### Requirement: 线条干扰面板

系统 SHALL 提供 6 个线条面板：水平线 1（1px 间距）、水平线 2（2px 间距）、垂直线 1、垂直线 2、正斜线 1（细）、正斜线 2（粗），使用 CSS `repeating-linear-gradient` 实现，黑白交替。

#### Scenario: 渲染水平线 1

- **WHEN** 用户进入"水平线 1"面板
- **THEN** 主画布 MUST 以 `repeating-linear-gradient(to bottom, #000 0 1px, #fff 1px 2px)` 全屏填充

### Requirement: 干扰图案面板

系统 SHALL 提供 5 个干扰图案面板：点阵、微型图案、闪烁方块、方形网格、彩色网格。

#### Scenario: 微型图案布局

- **WHEN** 用户进入"微型图案"面板
- **THEN** 主画布 MUST 在四个角 + 中心共 5 处渲染微型棋盘格区块（每块约 200×200px），其余区域为黑色

#### Scenario: 闪烁方块

- **WHEN** 用户进入"闪烁方块"面板
- **THEN** 系统 MUST 以约 2Hz 的频率在黑/白方块之间切换，rAF 驱动；并在进入此面板时一次性显示光敏警告提示

#### Scenario: 彩色网格

- **WHEN** 用户进入"彩色网格"面板
- **THEN** 主画布 MUST 以红、绿、蓝、青、洋红、黄循环填充 ~100×100 的等大方块铺满屏幕

### Requirement: 对比度面板（4×10 灰度矩阵）

系统 SHALL 提供"对比度(暗)"与"对比度(亮)" 2 个面板，各以 4 行 × 10 列共 40 个灰度色块布满画布，每个色块在中央显示其灰度百分比与 0-255 数值（文本颜色随背景自动反相）。

- **对比度(暗)**：灰度值集合从 `0` 起步密集排列于 `0-40`
- **对比度(亮)**：灰度值集合密集排列于 `175-254`，越靠后步长越小

#### Scenario: 显示数值标注

- **WHEN** 用户进入对比度面板
- **THEN** 每个色块 MUST 居中显示 `XX.XX%` 与 `(N/255)` 两行文本，文本颜色 MUST 与背景灰度做反相以确保可读

### Requirement: 渐变面板

系统 SHALL 提供 4 个渐变面板：白色渐变（黑→白）、红色渐变（黑→红）、绿色渐变（黑→绿）、蓝色渐变（黑→蓝），均为横向 256 级 `linear-gradient`，全宽全高。

#### Scenario: 渲染白色渐变

- **WHEN** 用户进入"白色渐变"面板
- **THEN** 主画布 MUST 以 `linear-gradient(to right, #000000, #ffffff)` 填充整个画布

### Requirement: 饱和度面板

系统 SHALL 提供"饱和度"面板：水平 HSL 色相过渡 `red → yellow → green → cyan → blue → magenta → red`，模拟彩虹条用于检查色彩饱和度过渡平滑性。

#### Scenario: 渲染饱和度条

- **WHEN** 用户进入"饱和度"面板
- **THEN** 主画布 MUST 以横向 HSL 色相渐变填满，至少包含 6 个等分色相节点

### Requirement: 屏幕信息面板

系统 SHALL 在"屏幕信息"面板汇总并展示当前屏幕信息：视口尺寸、屏幕物理尺寸、设备像素比、颜色深度、色域（srgb / p3 / rec2020）、HDR 支持、刷新率（rAF 估算）、触摸支持。

#### Scenario: 显示完整信息表

- **WHEN** 用户进入"屏幕信息"面板
- **THEN** 系统 MUST 以表格展示上述全部字段，对无法探测的字段显示"未知"

#### Scenario: 刷新率估算

- **WHEN** 屏幕信息面板挂载完成
- **THEN** 系统 MUST 通过 rAF 连续采集至少 120 帧，估算 Hz 值并舍入到 60/75/90/120/144/165/240 之一；同时显示原始平均帧间隔（ms）

#### Scenario: 窗口大小变化

- **WHEN** 用户拖动浏览器改变窗口大小
- **THEN** 系统 MUST 自动更新视口尺寸字段

### Requirement: 多点触控面板

系统 SHALL 在"多点触控"面板使用 `pointerdown / move / up` 事件追踪所有活动指针，并在 Canvas 上为每个 `pointerId` 用独立颜色绘制圆点与轨迹。

#### Scenario: 多指触摸

- **WHEN** 用户用多根手指同时触摸屏幕
- **THEN** 系统 MUST 为每个手指渲染独立颜色的接触点与移动轨迹

#### Scenario: 清空轨迹

- **WHEN** 用户按下 `R` 键或点击"清空"按钮
- **THEN** 系统 MUST 清除画布上的所有轨迹

### Requirement: 欢迎页与结束页

系统 SHALL 提供"欢迎页"（首次进入显示介绍、分辨率、光敏警告）与"测试结束"页（致谢与重新开始按钮）。

#### Scenario: 欢迎页内容

- **WHEN** 用户首次打开屏幕检测器
- **THEN** 欢迎页 MUST 显示：项目介绍、当前视口分辨率、屏幕物理分辨率与颜色深度、关闭浏览器缩放的提示、光敏警告

#### Scenario: 测试结束页

- **WHEN** 用户切换到"测试结束"
- **THEN** 主画布 MUST 显示一句致谢与"重新开始"按钮，点击后回到欢迎页

### Requirement: 样式与视觉规范

系统 SHALL 在非全屏区域（侧边栏、控制条、欢迎页、屏幕信息、对比度等含 UI 的面板）遵循项目明亮主题：`bg-white/95`、`border-zinc-200`、`rounded-xl`、`shadow-lg shadow-zinc-200/50`、`text-zinc-600/900`；侧边栏选中项使用 `bg-zinc-100` 高亮。在纯色/线条/图案等纯检测画布中 MUST 不应用任何圆角、阴影、边距，确保检测准确性。

#### Scenario: 检测画布无装饰

- **WHEN** 用户进入任一纯色、线条、图案、渐变、饱和度面板
- **THEN** 主画布 MUST 无圆角、无边距、无阴影；填充 100% 主画布区域
